⚙️ CÓMO FUNCIONA AHORA
─────────────────────────────────────────────────────────────────────────────────

1. DESARROLLO
   Desarrollador/Agente implementa feature en feature branch
   
2. COMMITS (Validado)
   Usa /commits-convencionales para validar formato
   Formato: tipo(ámbito): descripción
   Tipos: feat, fix, refactor, docs, test, build, etc.

3. TESTS (Obligatorio)
   dotnet test → todos pasan (xUnit, 42+ tests)
   
4. ARQUITECTURA (Validada)
   Usa /principios-arquitectura para respetar SOLID
   ViewModels desacoplados de WPF
   
5. PR (Revisión)
   Descripción menciona tipo y ámbito de cambios
   Revisor usa AGENTS.md checklist
   
6. MERGE a master
   Commits quedan documentados
   Versionamiento es automático y predecible
   
7. RELEASE (Manual)
   git tag -a v0.6.0 -m "Release 0.6.0"
   CHANGELOG.md se actualiza automáticamente
